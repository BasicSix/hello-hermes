🔍 Prometheus 配置

# prometheus.yml
global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'hermes'
    static_configs:
      - targets: ['localhost:9119']
    metrics_path: '/api/metrics'

  - job_name: 'system'
    static_configs:
      - targets: ['localhost:9090']

📈 Grafana 配置

数据源设置

{
  "name": "Prometheus",
  "type": "prometheus",
  "url": "http://localhost:9090"
}

仪表板指标

  • CPU使用率: rate(process_cpu_seconds_total[5m])
  • 内存使用: process_resident_memory_bytes
  • 响应时间: http_request_duration_seconds_bucket
  • 错误率: http_requests_total{status=~"5.."}

🔔 告警规则

groups:
  - name: hermes-alerts
    rules:
      - alert: HighCPUUsage
        expr: rate(process_cpu_seconds_total[5m]) > 0.8
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "High CPU usage on {{ $labels.instance }}"
          description: "CPU usage is above 80%"

🔗 Hermes 监控集成

Hermes Dashboard 内置 Prometheus 指标端点，开箱即用：

1️⃣ 启用指标暴露

# 启动时添加 --enable-metrics 参数
hermes dashboard --port 9119 --enable-metrics

2️⃣ 验证端点

curl http://localhost:9119/api/metrics

3️⃣ Prometheus 抓取配置（追加到 prometheus.yml）

  - job_name: 'hermes'
    static_configs:
      - targets: ['localhost:9119']
    metrics_path: '/api/metrics'
    # 可选：添加标签便于过滤
    params:
      format: ['prometheus']

✅ 验证成功标志: Prometheus UI 中可见 hermes_* 开头的指标（如 hermes_request_count_total）
