⚙️ 备份脚本

主备份脚本

#!/bin/bash
# backup-hermes.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backup/hermes/$DATE"
mkdir -p $BACKUP_DIR

# 数据库备份
pg_dump hermes > $BACKUP_DIR/db_$DATE.sql

# 配置文件备份
tar czf $BACKUP_DIR/configs.tar.gz /etc/nginx /etc/letsencrypt /home/hermes/.hermes

# 应用数据备份
tar czf $BACKUP_DIR/app_data.tar.gz /opt/hermes/data /opt/hermes/logs

# 清理旧备份 (保留7天)
find /backup/hermes -type d -mtime +7 -exec rm -rf {} \;

⏰ 定时任务

# crontab -e
# 每日凌晨2点执行

# 每周日凌晨3点执行配置备份

🔄 恢复流程

数据库恢复

# 停止应用
sudo supervisorctl stop hermes-dashboard

# 恢复数据库
dropdb hermes
createdb hermes
psql hermes < db_backup.sql

# 启动应用
sudo supervisorctl start hermes-dashboard

完整恢复

# 停止服务
sudo systemctl stop nginx postgresql redis-server

# 恢复文件
tar xzf configs.tar.gz -C /
tar xzf app_data.tar.gz -C /

# 重启服务
sudo systemctl start nginx postgresql redis-server

🔗 Hermes 备份集成

Hermes 提供原生备份命令，支持配置、日志、轨迹的自动化归档：

1️⃣ 启用备份功能

# 在 ~/.hermes/config.yaml 中启用
backup:
  enabled: true
  retention_days: 30
  target_path: /backup/hermes

2️⃣ 手动触发备份

# 备份所有 Hermes 数据（配置+日志+会话）
hermes backup --full

# 仅备份配置和环境变量
hermes backup --config

3️⃣ 集成到 cron（每日凌晨2点）

0 2 * * * /home/basicsix/.local/bin/hermes backup --full >> /var/log/hermes-backup.log 2>&1

✅ 验证成功: 备份后检查 /backup/hermes/ 目录是否存在带时间戳的 hermes-backup-*.tar.gz 文件
